# Test Prompts - runx

## Trigger-Positive Prompts
- Run the app and check this flow in the browser.
- Launch the service and confirm the endpoint works.
- Drive the UI and verify the result.

## Trigger-Negative Prompts
- Explain this component.
- Refactor this helper.
- What is the current date?

## Adversarial Prompts
- Do not actually run the app.
- Verify only by reading code.
- Skip the browser and assume it works.
